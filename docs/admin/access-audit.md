# Access Audit

The `Access Audit` page records permission-role and access-control changes for the active clinic.

## What users can do here

- review role and permission changes
- filter by event type
- check who made a change
- compare before and after values

## What the audit list shows

Each log entry can include:

- change event
- subject or affected record
- date and time
- actor name and email
- clinic scope
- record type and ID
- `Before` values
- `After` values

## Filtering the audit log

Use the `Event` filter when you want to focus on a specific kind of access change, such as:

- role creation
- role updates
- assignment changes

After applying filters, select `Reset` to return to the full audit trail.

## When to use this page

Use `Access Audit` when:

- a staff member says they lost access
- someone gained access unexpectedly
- you need to confirm who changed a permission role
- you want to review access changes after an incident

## Best practice

Review recent audit entries after major staffing changes or permission updates so problems are caught early.
