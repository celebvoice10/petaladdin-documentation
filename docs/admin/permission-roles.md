# Permission Roles

The `Permission Roles` page lets clinic administrators create reusable access bundles for staff accounts.

## What users can do here

- create clinic-specific permission roles
- review enabled package limits
- see assignable feature counts
- edit which features belong to each role

## What the role list shows

Each permission role can show:

- role name
- allowed features grouped by module
- quick edit action

## Before creating a role

The page also shows:

- active subscription package
- package clinic limit
- package staff limit
- number of assignable features

If no package features are available, the clinic cannot create permission roles until the package allows them.

## Creating a permission role

1. Open `Permission Roles`.
2. Select `Add Permission Role`.
3. Enter a role name.
4. Review the grouped feature list.
5. Use `All` or `None` inside a feature group when needed.
6. Select only the features the role should use.
7. Select `Save Permission Role`.

## How feature selection works

The role form includes:

- grouped permissions by module
- a feature search box
- a live count of selected permissions

This makes it easier to build focused roles such as:

- receptionist
- doctor
- billing executive
- inventory staff
- clinic admin

## Best practice

Create a small number of clear roles and reuse them. Too many near-duplicate roles make support and auditing harder.
