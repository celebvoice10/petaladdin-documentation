# Vaccinations

The `Vaccinations` module helps clinics track vaccine due dates, completion status, reminder planning, and prior vaccination history.

## What users can do here

- search vaccination records by vaccine, pet, parent, or phone
- filter by status
- sort by due date or vaccine name
- use `Add Vaccination`
- review prior vaccination history
- complete vaccination entries

## Vaccinations list screen

The list supports:

- search
- advanced filters
- status filtering
- sort order changes
- quick access to `Add Vaccination`

This screen is most useful for monitoring upcoming, overdue, and completed vaccinations.

## Vaccination plan workflow

The vaccination form is organized into two steps:

### 1. Vaccination Appointment

In this step, users:

- search and select the pet parent
- select the pet
- optionally link a consultation
- add visit notes

### 2. Vaccination Plan

In this step, users can add one or more vaccine rows and define:

- vaccine name
- dose date
- dose type
- status

The form also shows previous vaccination history for the selected pet.

## Previous history panel

The history panel helps users check:

- earlier vaccine names
- dose type
- completion status
- previous consultation links

This helps avoid duplicate or mistimed entries.

## Completing a vaccination

Depending on permissions and workflow, users may:

- save the plan for later
- mark the vaccination complete from the form

## When to use this page

Use it when:

- a vaccine is due
- the clinic wants to record a completed dose
- the team needs to check the pet's prior vaccine timeline

## Common mistakes to avoid

- selecting the wrong pet before entering the plan
- entering a new vaccine row without checking previous history
- saving a completed dose as pending by mistake

## Best practice

Always verify the pet and consultation context before saving. Vaccination records become much more useful when linked to the correct visit.
